---
title: Instance modes
description: How NeonFlux separates single and multi instance behavior.
---

NeonFlux has one deployment mode at a time. The bot writes that deployment config into the database during startup, and web reads the database value.

## Single instance

Single instance targets one configured community.

The dashboard does not show a community picker. If a user logs in without Manage Server permission for the configured community, the site shows a not-authorized screen for that community.

## Multi instance

Multi instance uses one bot token for many installed communities.

The dashboard lists only communities where:

- the user has Manage Server permission.
- the bot is installed.

If no manageable installed communities are found, invite flow is handled separately from this first docs foundation.
