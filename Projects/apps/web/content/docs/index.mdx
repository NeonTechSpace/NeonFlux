## Current status

The bot can run in two modes:

| Mode            | Meaning                                     |
| --------------- | ------------------------------------------- |
| Single instance | One configured community.                   |
| Multi instance  | One bot handles many installed communities. |

The dashboard uses Fluxer OAuth. Users only see communities they can manage, and server code re-checks access before meaningful changes. The selected-community overview shows tracked growth and message-activity charts after the bot starts collecting events. Invite attribution lives in the Invite Tracking category. Audit Events supports field-scoped search and scroll-loaded history.

The Messaging category can send plain text and Fluxer embed payloads. Builder mode applies the sidebar color directly from the color picker; Advanced JSON remains available for custom embed arrays.

Moderation has live case, warning, reason, and note commands. Punishments such as ban, kick, timeout, purge, and automod are still planned.

## Dashboard routes

| Route          | Purpose                 |
| -------------- | ----------------------- |
| `/`            | Public homepage.        |
| `/docs/*`      | Public documentation.   |
| `/auth/*`      | Fluxer OAuth.           |
| `/dashboard/*` | Protected dashboard UI. |

Open `/dashboard` to choose a community. A specific community is managed at `/dashboard/{guildId}`.
