NeonFlux currently listens to normal bot messages. Slash commands are not the supported command path right now.

## Custom status

After the gateway is ready, NeonFlux shows `DEFCON 1` or `DEFCON 2` as the custom status when the deployment-level
effective DEFCON is locked down. In normal mode, DEFCON 3, NeonFlux uses `FLUXER_BOT_CUSTOM_STATUS` from the bot
environment. Leave it blank in normal mode for no custom status.

```txt title="Example custom status"
Artron energy is binding to NeonFlux, allowing travel throughout all of existence.
```

<Callout title='Current command model' type='info'>
    Prefix commands are the active path. The default prefix is `!`, and communities can change it with a guarded mention
    command or from the selected community dashboard.
</Callout>

## Command prefix

The default command prefix is `!`.

Server owners, or users with permission to change guarded bot settings, can change the prefix by mentioning the bot:

```txt title="Prefix change"
@NeonFlux prefix ?
```

The mention-based prefix command works regardless of the current prefix.
If the requested prefix already matches the current prefix, NeonFlux reports that nothing changed instead of saving it again.

The prefix can also be changed from the command prefix panel on the selected community dashboard.
When that panel is open, prefix changes from bot commands or another dashboard tab update live without interval polling.

Valid prefixes are 1-3 visible characters. The first character must be an allowed symbol or punctuation mark. After the
first character, letters and numbers are allowed.

Do not use `/`, `@`, `#`, `<`, `>`, or `:` because those characters overlap with Fluxer slash commands, mentions,
channels, and other platform syntax. Prefixes also cannot contain spaces, control characters, or invisible characters.

## Ping command

`ping` uses the current community prefix. With the default prefix, `!ping` replies:

```txt title="Ping response"
Yes, I'm here, and no, I don't pong
```

This command is intentionally small. It exists to prove that message routing, prefix parsing, DEFCON checks, and channel replies work before larger bot features are added.

## Help command

`help` uses the current community prefix and lists available command pages. With the default prefix, `!help` shows the
current pages:

```txt title="Help pages"
!help general
!help settings
```

Users can open a page directly with `!help general` or `!help settings`. Mentioning the bot with `@NeonFlux help` opens
the same help entrypoint. Help pages are plain text right now; button navigation is not supported until Fluxer message
components are wrapped.

## Mention behavior

If someone mentions only the bot with no useful context, NeonFlux replies with escalating reminders.

<Steps>
    <Step>First empty mention: the bot asks the user to say what they need instead of only calling its name.</Step>
    <Step>Repeated empty mentions: the bot gets shorter and less patient.</Step>
    <Step>After enough empty mentions: the bot ignores that user for 5 minutes.</Step>
    <Step>After the cooldown: the next empty mention gets one final reminder and starts the cooldown again.</Step>
</Steps>

Messages that mention the bot with real context are ignored by this presence feature. The bot expects the user to say what they need.

## DEFCON effect

DEFCON affects presence replies, public help, and guarded prefix changes.

| DEFCON state     | Presence, help, and prefix behavior                                             |
| ---------------- | ------------------------------------------------------------------------------- |
| Normal           | Presence and help can run, and Manage Server users can change the prefix.       |
| Locked down      | Guarded prefix changes are server-owner only. Public presence and help can run. |
| Highest lockdown | The bot listens only to the server owner unless a category is explicitly freed. |

<Callout title='Current configurability' type='warn'>
    Prefix can be changed through the mention command or the selected community dashboard. Mention-response and DEFCON
    controls are not configurable in the dashboard yet.
</Callout>
